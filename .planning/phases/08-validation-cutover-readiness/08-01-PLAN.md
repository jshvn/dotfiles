<objective>
Refactor `taskfiles/links.yml` per D-08 to introduce a single top-level `EXPECTED_TARGETS` var listing every symlink target owned by links.yml, retire all 5 cmds-spanning `{{if}}...{{end}}` wrappers (which currently produce the `template: :1: unexpected EOF` warning documented in `07-VERIFICATION.md` lines 124-127), flip the 2 bare `deps: [manifest:resolve]` refs to the leading-colon root-namespace form, and rewrite `links:validate` as a single shell-block with a failures counter so it exits non-zero on any broken symlink (closing the `_:check-link` always-exits-0 gap confirmed in helpers.yml lines 48-73).

Purpose: This is the structural prerequisite for P2 (root `task validate` aggregator). Without the rewrite, the aggregator's call to `links:validate` cannot detect failures via exit code, and the EOF warning leaks into validate output on every run. The EXPECTED_TARGETS var becomes the single source of truth that P3's `links:reconcile` orphan detector iterates.

Output: A single modified file (`taskfiles/links.yml`) that retains all current behavior (links:all + zsh + antidote + claude + configs + identity:install delegation) while exposing the EXPECTED_TARGETS catalog and a correctly-exiting validate task.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user shell → `task links:validate` | command-line invocation; no untrusted input enters the task body |
| EXPECTED_TARGETS var → shell loop | feature-gate ternary render produces empty lines when off; consumer must guard against blank-line injection via `[[ -z "$target" ]] && continue` |
| symlink target → `readlink -f` resolution | symlink target paths are filesystem state; not user input; bounded to {{.DOTFILEDIR}} prefix comparison only (no `eval`, no shell expansion of resolved paths) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | EXPECTED_TARGETS var rendering | mitigate | Inline-ternary form `{{if cond}}path{{end}}` produces deterministic empty-line on feature-off; consumer skips via `[[ -z ]] && continue`; no shell metacharacters in any TARGET path (all under XDG dirs) |
| T-08-02 | Information disclosure | `readlink -f` on user-owned symlinks | accept | Targets are bounded to XDG_CONFIG_HOME and ZDOTDIR which are user-owned; resolving them leaks no information beyond what `ls -la` already shows |
| T-08-03 | Denial of service | EXPECTED_TARGETS iteration | accept | List is bounded to 27 entries; iteration is O(n) shell loop; worst-case runtime is sub-second on the slowest target machine (server-1 over SSH) |
| T-08-SC | Tampering | npm/pip/cargo installs | n/a | This plan installs no packages — pure taskfile refactor |
</threat_model>
