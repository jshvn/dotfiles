<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator (you) → four target machines | operator must have physical/SSH access; mistakes during cutover could leave a machine in a hybrid v1/v2 state |
| docs/CUTOVER.md state table → operator memory of cutover progress | the table is the only source of truth for cutover status across the multi-week window |
| v1 archive operation → production data on all four machines | mistaken `rm -rf` instead of `mv` would destroy v1 fallback path; CUTV-06 explicitly mandates archive-not-delete |
| cutover-ack writer per machine → install gate enforcement | wrong machine name in `task cutover:ack -- <name>` is rejected by the regex + active-machine match (P3 Task 2 mitigation) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-19 | Tampering | premature v1 archive before all machines cut over | mitigate | Task 5 explicit pre-condition gate: all four rows must show `cut-over` AND `days-on-v2 >= 7`; archive-not-delete preserved per CUTV-06 |
| T-08-20 | Repudiation | rollback during soak loses track of cutover state | mitigate | docs/MIGRATION.md § "Rollback" step 4 mandates writing back to per-machine notes column; status returns to `planning` so the soak counter resets cleanly |
| T-08-21 | Denial of service | work-laptop cutover during high-work-disruption period | accept | operator scheduling decision; mitigated by ordering work-laptop LAST after personal + both servers soak successfully |
| T-08-22 | Information disclosure | docs/CUTOVER.md committed to git with operational state | accept | per-machine names, statuses, dates are not secrets; same level of disclosure as commit history |
| T-08-23 | Elevation of privilege | `task install` on a wrong machine due to swapped state file | mitigate | cutover:ack regex + active-machine match enforced at the gate (P3 Task 2); install precondition (cutover_gate_check) blocks if names mismatch |
| T-08-SC | Tampering | npm/pip/cargo installs | n/a | P6 is operator-driven runbook; no new package installs beyond what `task install` already does |
</threat_model>
