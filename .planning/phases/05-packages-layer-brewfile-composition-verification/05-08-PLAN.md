                 - Use jq to look up the cask record: jq -r --arg t "$name" '.casks[] | select(.token == $t)' <<<"$brew_info_json"
                 - If no record (cask declared but not installed per brew info): cross + increment failures
                 - If record exists, walk .artifacts[]: each artifact is an object like
                   {"app": ["Slack.app"]}, {"binary": ["op"]}, {"pkg": ["docker.pkg"]}, etc.
                   For each artifact kind:
                     * "app" -> assert /Applications/<filename>.app exists (test -d); cross if missing
                     * "binary" -> assert command -v <basename> resolves (binary-only casks); cross if missing

               For each formula declared in {{.COMPOSED_BREWFILE}} via `^brew '<name>'` regex:
                 - Use jq to look up the formula record: jq -r --arg n "$name" '.formulae[] | select(.name == $n)' <<<"$brew_info_json"
                 - If no record (formula declared but not installed): cross + increment failures
                 - If record exists, assert .installed[0].linked_keg is non-null
                 - ALSO: if the source bundle line carries `# verify: <bin>` (D-05 escape hatch),

        - mas missing on PATH: WARN (mas is in core.rb so should always be installed); skip MAS section gracefully

    Summary:
      - If failures > 0: cross "<n> package(s) failed verify"; exit "<n>"
      - Else: success "all declared packages verified"; exit 0
