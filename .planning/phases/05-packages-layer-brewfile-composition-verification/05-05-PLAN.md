<objective>
Wire `taskfiles/packages.yml` (from Plan 04) into the root install pipeline. Three discrete
edits to `Taskfile.yml`:

  1. Rename the `brew:` include key to `packages:` and point it at `./taskfiles/packages.yml`
     (replacing the `./taskfiles/brew-stub.yml` reference). Add explicit `vars:` forwarding for
     `DOTFILEDIR`, `XDG_STATE_HOME`, `XDG_CACHE_HOME` (the new dimension P5 introduces), and
     `DOTFILES_MESSAGES` -- mirroring the `manifest:` and `identity:` include blocks.

  2. Replace `task: brew:install` with `task: packages:install` in the root install task's
     `cmds:` list.

  3. Add `task: packages:verify` to the root install task's `cmds:` list, positioned AFTER
     `task: macos:shell` and BEFORE the trailing `success "install complete"` line. Per D-10
     and ROADMAP success criterion #6 (verify is the install pipeline's final step before
     the success message; a verify failure hard-fails the install with exit 1).

Purpose: Implements VRFY-04 (verify runs in install's final step). Closes the "silent install
failure" gap that VRFY-04 names -- a successful `brew bundle install` followed by a missing
binary or `.app` no longer reports success at the install level.

Output: One updated root `Taskfile.yml`. No taskfile changes elsewhere; no manifest changes;
no bundle changes; no composer changes.
</objective>
