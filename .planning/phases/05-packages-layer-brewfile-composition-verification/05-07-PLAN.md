<objective>
Strip the per-line `# verify:` comment convention and the per-cask `{ name, verify }` schema field
from every source artifact in service of the brew-info-driven verify pivot decided 2026-05-15 in
05-UAT.md Gap 2. After this plan executes, the active machine's composed Brewfile (`$XDG_CACHE_HOME/dotfiles/Brewfile`)
emits casks without `# verify:` annotations; every machine TOML's `extra_packages.casks` carries
name-only objects; the schema docs reflect verify-optional; the resolver dedupes cleanly on `.name`
alone; the composer's jq cask-emit filter drops the verify concatenation; the existing 11/11
manifest fixtures continue to pass.

Purpose: The brew-info-driven verify (Plan 05-08) is the authoritative artifact-presence check
post-pivot. The per-line `# verify:` comment metadata is now redundant noise that re-breaks on the
next upstream rename (the failure class Gap 2 surfaced). Removing the redundant metadata is a
prerequisite for Plan 05-08 -- the new `packages:verify` body cannot smoke-test cleanly against a
composed Brewfile that still emits `# verify:` cask comments with stale targets like
`/Applications/NVIDIA GeForce NOW.app`.

Output: 11 files edited (2 bundle files, 5 manifest TOMLs, 2 install scripts, 2 docs). No new
files; no file deletions. The MAS path (extra_packages.mas[].name) is UNCHANGED -- App Store has
no `brew info` equivalent, so the `name` field doubles as the verify target (D-06 preserved).
</objective>
