<objective>
Build the per-machine Brewfile composer (`install/compose-brewfile.zsh`) AND extend
`install/resolver.zsh`'s Pass 2 merge logic to handle the typed-bucket extras schema
from Plan 02. The composer reads `resolved.json`, concatenates the declared bundles
(`packages/<name>.rb` files from Plan 01), appends typed-extras as Ruby DSL lines, and
writes atomically to `$XDG_CACHE_HOME/dotfiles/Brewfile`.

Purpose: Implements PKGS-02 (composes the per-machine Brewfile from manifest bundles),
PKGS-03 (the composed file is the input to `brew bundle check` in Plan 04's status block),
and the resolver side of PKGS-04 (per-sub-array concat+dedupe at resolve time). Without
this plan, Plan 02's typed-bucket TOMLs would produce a malformed `resolved.json` and Plan 04
would have nothing to feed `brew bundle install`.

Output: One new script (`install/compose-brewfile.zsh`) + one update to `install/resolver.zsh`
(Pass 2 only). No taskfile changes (Plan 04 invokes the composer); no manifest changes
(Plan 02 owns those); no bundle file changes (Plan 01 owns those).
</objective>

  Legacy (lines 316-320):
    Compute `add | unique` over a single flat array .packages.brew.extra_packages
  New (D-03 typed-bucket):
    For each of formulae / casks / mas:
      def_array  = .packages.brew.extra_packages.<bucket> from defaults.toml

    .packages.brew.extra_packages.<bucket>
