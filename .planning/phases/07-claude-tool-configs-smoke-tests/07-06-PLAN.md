<objective>
Extend `taskfiles/links.yml` with two new sub-tasks (`claude:` and `configs:`), wire them into the `all:` aggregator, and extend the `validate:` task to include strict-mode (`SOURCE`-passing) check-link entries for every new symlink. This is the final integration plan for Phase 7: it brings together Plan 02's hardened helpers, Plan 03's `taskfiles/claude.yml`, and Plan 05's `configs/<tool>/` content into a single executable install pipeline.

Purpose: CLDE-01 + TOOL-01 + TOOL-02 specify that Claude config tree and tool configs deploy via `taskfiles/links.yml` using `_:safe-link`. TOOL-04 specifies that `_:check-link` enforces all three conditions including the `readlink -f` equality check; this plan is where the existing P3/P4 callers also gain the `SOURCE` arg (per CONTEXT.md Claude's Discretion "Retrofit decision ... recommend yes"). The `all:` aggregator extension is what causes `task install` (which calls `task links:all`) to deploy the new symlinks at install time.

Output: A single extended `taskfiles/links.yml` carrying ~20 new lines for claude + configs entries, with template-var-only status blocks, kebab-case index gating, and strict-mode validate entries. No other file is modified by this plan.
</objective>
