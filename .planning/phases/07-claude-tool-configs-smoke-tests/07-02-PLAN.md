<objective>
Harden the two symlink helpers in `taskfiles/helpers.yml` and fix the v1 `local`-at-script-scope shellcheck error in `claude/hooks/agent-transparency.zsh`. All three changes are Wave-1 prerequisites that subsequent Phase-7 plans depend on: Plan 06 (links.yml extension) calls the hardened `_:safe-link` for claude + configs entries and passes `SOURCE` to `_:check-link`; Plan 04 (smoke tests) pipes a synthetic JSON payload through the rewritten `agent-transparency.zsh` and asserts it does not crash.

Purpose: TOOL-03 closes the v1->v2 cutover safety gap (real file at v2 symlink destination). TOOL-04 closes the "symlink exists but points to stale path after refactor" gap. CLDE-02's agent-transparency rewrite closes the LINT-07 / shellcheck regression class that survives by accident because zsh-as-script doesn't fail on script-scope `local` at parse time but does at strict-mode `set -u` eval (script currently still works because `set -euo pipefail` doesn't trap `local` outside a function — but shellcheck flags it and the smoke test invocation pattern is sensitive).

Output: Backward-compatible helpers with strict-mode hooks for new callers, plus a function-wrapped agent-transparency hook that conforms to v2 conventions and the same shape as `os/defaults/<concern>.zsh` from P6.
</objective>
