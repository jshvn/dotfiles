<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dead-code removal → repo state | A removal that drops a live-but-niche symbol breaks behavior silently if grep misses it. |
| orphan fixture removal → test suite | Removing a fixture that secretly tested live code regresses the test suite invisibly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-03-01 | Tampering | Class A interactive function/alias mistakenly classified Class B | mitigate | D-08 Class A allowlist is path-based and EXPLICIT: only `shell/functions/*.zsh` and `shell/aliases/*.zsh` are preserved. The executor verifies every removal candidate's path is NOT under these globs before grep-checking. |
| T-13-03-02 | Tampering | grep false-negative on a live symbol (e.g., dynamic dispatch via $varname) | mitigate | The grep command uses word-boundary anchors `\b<symbol>\b` to prevent substring false-positives, but dynamic dispatch can defeat grep entirely. For every removal, the executor additionally runs `task install --dry` AND `task validate` in dry-run after the removal commit; failure surfaces a missed callsite as a runtime break, not a silent regression. |
| T-13-03-03 | Repudiation | removal commit without evidence | mitigate | Every removal commit body lists each dropped symbol with the exact grep command run + the result (`0 hits` or the actual hits — though if hits exist the symbol is NOT dead and the removal is reverted before commit). |
| T-13-03-04 | Denial of Service | fixture removal breaks `task test` | mitigate | After every fixture removal, `task test` is run. If it exits non-zero, the fixture was not orphan — the removal is reverted and the fixture's role re-investigated. |
</threat_model>

      git grep -nE '\b<symbol>\b' -- '*.zsh' '*.yml' '*.toml' '*.rb' 'Brewfile*' '*.md' ':!.planning/' ':!.git/'

    File: <path>:<line>
    Type: task / helper / function / branch / fixture

    Grep result: 0 hits | <N> hits (NOT dead)
    Action: remove | keep
    REVIEW.md row: line <N>

    - Fixture references a file/symbol that no longer exists in v2 → ORPHAN → mark for removal.
    - Fixture references a Phase 12-renamed symbol → UPDATE the fixture in place (rename inside the fixture); do not remove.
