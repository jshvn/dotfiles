<objective>
Rewrite the root `bootstrap.zsh` to be the hardened v2 trust-anchor acquisition script per CONTEXT.md decisions D-01 through D-05. The script's three jobs are: (1) install brew if missing (with audit log + 3s abort window per D-01), (2) install go-task and yq via `brew install` (D-02), (3) print a next-step hint listing available machines (D-03). Bootstrap is tools-only — it does NOT invoke `task setup` and does NOT take a machine-name argument.

Note on cutover-gate: per RESEARCH §3.4 / §3.6, bootstrap eventually sources `install/cutover-gate.zsh` and calls `cutover_gate_check` BEFORE the next-step hint. That gate file is built in Plan 02-04. To keep Wave 1 plans independent, this plan does NOT add the gate sourcing (Plan 02-04 will do that as a follow-up edit). Plan 02-04 owns the file-modify because the gate file is the output of that plan.

Purpose: BTSP-01..03 ship together as a single bootstrap rewrite. BTSP-04 (`task setup -- <name>` writes machine state) is already implemented in P1's `taskfiles/manifest.yml` — bootstrap merely references it in the next-step hint.

Output: A single rewritten `bootstrap.zsh` at the repo root. v1's `bootstrap.zsh` is overwritten in place per D-06 (branch-only isolation; v1 stays on master).
</objective>
