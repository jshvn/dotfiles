<!-- Merge rules every fixture must encode (RESEARCH §4 + §5, CONTEXT D-06). -->
<!-- These are the hand-computation contract — fixture authors must apply each rule exactly. -->

Merge rules (D-06):
- Maps (tables): deep-merge; machine wins on conflict; sibling keys preserved.
- Scalars (string/number/bool): machine replaces defaults.
- Arrays: replaced wholesale (machine wins) — EXCEPT `packages.brew.extra_packages`.
- `packages.brew.extra_packages`: concatenated then deduplicated (union; machine values first, defaults values not in machine appended).

Verification flow (each fixture):
- Fixture 1..6: `task manifest:test` runs `diff <(jq -S . actual) <(jq -S . expected)` and expects zero diff.
- Negative fixtures: resolver/validator must exit non-zero with stderr containing a specific error fragment.

Reference fixture-06 union computation (worked manually):
- defaults.extra_packages = ["jq", "yq"]
- machine.extra_packages = ["docker-desktop", "jq"]
- Union (machine-first, dedupe): ["docker-desktop", "jq", "yq"]

Reference fixture-04 nested-table shape (RESEARCH §4.1):
- defaults declares [a.b.c.d] with shared="defaults-value", default_only=true, e={ deep_nested=true }
- machine declares [a.b.c.d] with shared="machine-value", machine_only=true, e={ deep_nested_override="yes" }, f={ totally_new=true }
- Expected merged: shared="machine-value", default_only=true (preserved), machine_only=true, e={ deep_nested=true, deep_nested_override="yes" } (deep-merged), f={ totally_new=true }

JSON output convention (matches RESEARCH §4 verifications):
- TOML scalar `true` → JSON `true`
- TOML scalar `"darwin"` → JSON `"darwin"`
- TOML array `["core", "gui"]` → JSON `["core", "gui"]` (order preserved)
- TOML table `[features]` → JSON `"features": { ... }`
- jq -S sorts keys, so authoring order in expected.json does not matter as long as content matches.
</interfaces>
