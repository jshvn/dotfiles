<objective>
Build the deep-merge resolver (`install/resolver.zsh`) plus the four real machine manifests and the shared defaults.toml. This is the keystone implementation: every downstream phase reads its inputs from the resolved.json this script produces.

The resolver supports two modes:
1. **Resolve mode** (default invocation): reads `$XDG_STATE_HOME/dotfiles/machine`, merges defaults + that machine's TOML, writes resolved.json atomically.
2. **Validate-only mode** (`--validate-only --machine <name>`): runs the D-03 required-field validator + D-01 os-value check + D-04 unknown-key warnings against a specified manifest, exits non-zero on hard errors, exits 0 with stderr warnings on unknown keys.

The four machine manifests follow the schema sketch from RESEARCH §3.2, parameterized for `personal-laptop`, `work-laptop`, `server-1`, `server-2`.

This plan depends on Plan 01 fixtures existing (no direct file dependency, but Plan 03 in Wave 2 will diff this resolver's output against Plan 01's expected.json files — so the resolver must be correct against those fixtures before Plan 03 wires the test driver).

Purpose: Make the manifest layer real — TOML in, JSON out, schema enforced.
Output: One zsh executable + five TOMLs + two READMEs.
</objective>

<!-- Resolver CLI surface, exit codes, and pipeline contract. -->
<!-- Plan 03 (taskfiles/manifest.yml) invokes resolver.zsh; this contract is binding on both. -->

Invocation surface:
  zsh install/resolver.zsh                              # resolve mode: reads state file, writes resolved.json
  zsh install/resolver.zsh --validate-only \
      --machine <name>                                  # validate only; no resolved.json write
  zsh install/resolver.zsh --machine <name> --stdout    # ad-hoc resolve to stdout (used by manifest:show -- --machine)

Required env at invocation:
  DOTFILEDIR        — exported by Taskfile.yml vars block (caller responsibility).
  XDG_STATE_HOME    — fallback to $HOME/.local/state when unset.

Exit codes:
  0   — success (resolve OK; validation passed; unknown-key warnings allowed)
  1   — D-16 missing state file, OR machine TOML not found, OR --validate-only found a required-field violation, OR platform.os != "darwin"
  2   — internal error (yq/jq invocation failed, write failed)

Reusable functions exposed by resolver.zsh (sourced indirectly via taskfiles):
  resolve_manifest <defaults_path> <machine_path> <out_path>   — three-pass merge → atomic write
  validate_manifest <machine_path>                              — required-field set + os check + unknown-key warnings
  list_available_machines                                       — space-separated machine name list from manifests/machines/*.toml

Three-pass merge pipeline (RESEARCH §4.2 + §5):
  Pass 1: yq eval-all '. as $i ireduce ({}; . * $i)' $DEFAULTS $MACHINE -o json
  Pass 2: union+dedupe packages.brew.extra_packages from both source files using yq -o=json + jq -s 'add | unique'
  Pass 3: backfill platform.arch via uname -m if machine TOML lacks it
  Final:  jq --argjson extras "$union_extras" --arg arch "$arch" '... assignments ...' → tmp → mv to resolved.json

Required-field validator (RESEARCH §3.3, D-03):
  - meta.description must be a non-empty string (yq has() + tag !!str)
  - platform.os must equal "darwin" (D-01)
  - features must be a table (tag !!map), may be empty {}
  - packages.brew.bundles must be a non-empty array (tag !!seq, length >= 1) that includes "core"
  - identity.git ∈ {"personal","work","none"}
  - identity.ssh ∈ {"personal","work","none"}

Unknown-key warnings (D-04, RESEARCH §3.4):
  - Whitelist of expected paths: schema_version, meta.*, platform.{os,arch}, features.* (any subkey allowed), packages.brew.{bundles,extra_packages}, identity.{git,ssh}
  - For each leaf path in the machine TOML not under the whitelist, emit `unknown key: <path> at <file>:<line?>` to stderr, exit 0.

Atomic-write contract (RESEARCH §13 Q5):
  - Write to `${OUT}.XXXXXX` via mktemp, then `mv` to final path
  - Never leave partial JSON visible to readers

Machine manifest values (PROJECT.md "machines" + CONTEXT.md "Reference machine list"):
  personal-laptop — full GUI + dev + personal; identity = personal
  work-laptop    — GUI + dev + work bundle; identity = work; one-password-ssh = true
  server-1       — core only; no GUI feature flags; identity = none; one-password-ssh = false
  server-2       — core only; no GUI feature flags; identity = none; one-password-ssh = false

Messages library (Pattern 2):
  Source guard:  [[ -n "${DOTFILES_MESSAGES_LOADED:-}" ]] || source "${DOTFILEDIR}/install/messages.zsh"
  Available: info, success, warn, error (writes to stderr), check, cross, header, step
