Structural fix (option 2 from debug doc, recommended):
  Replace the workstation gitdir literals with a find that locates the .git
  directory of any subrepo under ~/git/<identity>/. Use the parent of the
  located .git as the probe source. Replace the `[[ -d ]]` guard with
  `git -C "$probe_dir" rev-parse --is-inside-work-tree` so we never run
  `git config user.email` from a non-repo location. If no probe repo
  exists, skip cleanly (info message + exit 0) -- this preserves the
  fresh-machine UX where ~/git/personal/ may not have any clones yet.

  Server branch (identity=server-1|server-2): UNCHANGED. Server identities
  use unconditional `[include] path = server-include.config` (see
  identity/git/config:22-23) which fires regardless of repo context, and
  $HOME is always a valid directory; the existing code is correct.

Key behaviour of `find ~/git/<id> -maxdepth 2 -name .git -type d -print -quit`:
  - -maxdepth 2 means: ~/git/<id>/ itself is depth 0; its immediate
    children are depth 1; their immediate children are depth 2. So
    ~/git/personal/dotfiles/.git matches at depth 2 (the .git dir
    itself, parent of dotfiles, parent of personal). This catches the
    standard one-level-deep clone layout.
  - -print -quit makes find emit the FIRST match and exit; we never need
    a list, just a single probe source. Stable order is not guaranteed
    across find versions but any matching repo serves equally as a probe.
  - If no .git dirs exist (or ~/git/<id> doesn't exist), find prints
    nothing and exits 0 (no-match is not an error). The shell test
    `[[ -z "$dotgit" ]]` distinguishes the no-probe case.
  - find runs against $HOME paths only; no cwd-sensitivity. Apostrophes
    in nested paths (e.g., joshs-air) are not a concern under -name .git
    because we match a literal directory name.
  - -type d guards against the rare case where a submodule's .git is a
    file (gitlink). A submodule probe wouldn't be wrong, but using an
    actual top-level repo gives the cleanest test.

Why `git rev-parse --is-inside-work-tree` is the right work-tree guard:
  - Returns "true" + exit 0 when run from inside a real work tree.
  - Returns non-zero exit when run from a non-repo or a bare-repo location.
  - This is git's own canonical "am I inside a work tree" probe, used by
    countless prompt scripts and tooling. Catches the failure mode that
    `[[ -d "$gitdir" ]]` missed (a directory exists but is not a repo).

XDG/HOMEBREW conventions (verified in scope):
  - $HOME is the only env var the validate:git task body needs; it is
    always set in zsh under -u as long as the user has a valid login.
  - Homebrew prefix not relevant -- this task uses only system git.
  - find is the BSD find on macOS by default; -maxdepth, -name, -type, -print,
    -quit are all in the BSD find baseline (no GNU-only flags used).

      validate:git:
        desc: "validate (b): git user.email matches manifest identity"
        internal: true
        cmds:
          - |
            {{.DOTFILES_MESSAGES}}
            identity="{{.MANIFEST.identity.git}}"

            # Resolve the probe source per identity. Workstation identities
            # (personal, work) require a REAL git repo under ~/git/<identity>/
            # so the [includeIf gitdir/i:~/git/<identity>/] block in
            # identity/git/config can fire and load the per-identity overlay.
            # Server identities use unconditional [include] server-include.config
            # so they probe from $HOME directly.
            #
            # WR-* fix (UAT gap 4): the previous implementation used the
            # parent directory $HOME/git/<identity> as the probe source,
            # which is not a git repo, so the gitdir-conditioned includeIf
            # never fired and `git config user.email` returned empty.
            case "$identity" in
              personal|work)
                root="$HOME/git/$identity"
                expected_email=$(git config -f "{{.DOTFILEDIR}}/identity/git/identities/$identity" user.email 2>/dev/null || echo "")
                # Locate the first .git directory under the identity root
                # (depth 2 = standard clone layout: $root/<repo>/.git).
                # -print -quit returns the FIRST match and exits; no-match
                # is silent and exit 0.
                dotgit=$(find "$root" -maxdepth 2 -name .git -type d -print -quit 2>/dev/null || true)
                if [[ -z "$dotgit" ]]; then
                  info "no git repo found under $root -- skipping email assertion"
                  exit 0
                fi
                probe_dir=$(dirname "$dotgit")
                ;;
              server-1|server-2)
                probe_dir="$HOME"
                expected_email=$(git config -f "{{.DOTFILEDIR}}/identity/git/identities/${identity}" user.email 2>/dev/null || echo "")
                ;;
              none)
                info "identity.git = none -- skipping email assertion"
                exit 0
                ;;
              *)
                cross "identity.git = $identity is unexpected; skipping"
                exit 0
                ;;
            esac

            if [[ -z "$expected_email" ]]; then
              info "expected git user.email for identity.git=$identity is empty -- skipping"
              exit 0
            fi

            # Verify $probe_dir is actually a git work tree before running
            # config queries. Replaces the prior `[[ -d "$gitdir" ]]` guard
            # which only checked directory existence, not git-repo-ness.
            # Server branch ($probe_dir = $HOME) typically returns true on a
            # developer machine; on a fresh server with no git history,
            # rev-parse fails non-zero and we skip cleanly.
            if ! git -C "$probe_dir" rev-parse --is-inside-work-tree >/dev/null 2>&1; then
              info "$probe_dir is not a git work tree -- skipping email assertion"
              exit 0
            fi

            actual_email=$(git -C "$probe_dir" config user.email 2>/dev/null || echo "")
            if [[ "$actual_email" = "$expected_email" ]]; then
              check "git user.email matches identity ($expected_email)"
            else
              cross "git user.email mismatch: expected '$expected_email', got '$actual_email'"
              exit 1
            fi
